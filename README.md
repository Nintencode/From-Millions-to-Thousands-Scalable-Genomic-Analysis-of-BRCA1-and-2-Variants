# From Millions to Thousands: Scalable Genomic Analysis of BRCA1 and 2 Variants
## Short Overview:
This project analyzes DNA variation in BRCA1/BRCA2 variants for Latin American ancestry participants from the All of Us Research Program. Using computational workflows and prediction models, we investigate variant pathogenicity to address gaps in genetic risk interpretation in underrepresented populations.

## Project Summary:
Mutations in BRCA1 and BRCA2 are strongly associated with an increased risk of hereditary cancers, particularly breast and ovarian cancer. Accurate classification of genetic variants in these genes is critical for identifying individuals who may carry elevated cancer risk and guiding clinical decision-making.

Many genetic studies, however, have historically focused on populations of European ancestry, leaving many other populations underrepresented in genomic research. This underrepresentation creates gaps in variant interpretation and limits the accuracy of pathogenicity prediction for individuals from these populations.

To address this challenge, this project uses genomic data from the All of Us Research Program, which provides a large and diverse dataset designed to improve representation in biomedical research. The analysis specifically focuses on participants with Latin American ancestry to better understand the distribution and predicted impact of BRCA1 and BRCA2 variants within this population.

A computational workflow was developed to extract, clean, and process variant data from the All of Us Researcher Workbench. Identified BRCA1/2 variants were then evaluated using established variant prediction models such as PolyPhen-2 and REVEL. Additional machine learning approaches were explored to analyze variant characteristics and assess patterns in predicted pathogenicity.

By combining diverse genomic data with computational prediction methods, this project aims to contribute to improved variant interpretation and support more equitable genomic risk assessment across underrepresented populations.

## Script Summaries

### All of Us V8 BRCA 1 Data Extraction and Annotation Integration
This script establishes the foundational data extraction workflow for the project by identifying and preparing BRCA1 variants from the All of Us Research Program V8 whole genome sequencing dataset. Using the Hail genomic analysis framework, the pipeline loads the large-scale variant data as a matrix table, filters variants to the genomic interval corresponding to BRCA1, and integrates participant ancestry information provided by the All of Us database. The script then imports the Variant Annotation Table (VAT) and filters it to canonical BRCA1 transcripts, retaining key annotation variables such as variant consequence, ClinVar classification, and REVEL prediction scores. Finally, these annotation variables are joined with the filtered genomic variant matrix table to create a unified dataset linking participant-level genomic variants with functional and clinical annotations. This processed dataset serves as the primary input for downstream analyses in the project, including variant characterization and machine learning models used to evaluate pathogenicity patterns in BRCA1 and BRCA2 variants among individuals with Latin American ancestry.

[All of Us V8 BRCA 1 Data Extraction and Annotation Integration link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/All%20of%20Us%20V8%20BRCA%201%20Data%20Extraction%20and%20Annotation%20Integration.ipynb)

### All of Us V8 BRCA 2 Data Extraction and Annotation Integration
This script follows the same workflow used in the BRCA1 data extraction but applies it to BRCA2 DNA variants. The All of Us V8 whole genome sequencing dataset is filtered again to include only the region of DNA where the BRCA2 gene is located. The same important annotation variables such as ancestry, consequence, and Clinvar classification is integrated into the BRCA2 data as well. The final dataset mirrors the structure of the BRCA1 data and serves as the other half of our input for later variant analysis and machine learning models in the project.

[All of Us V8 BRCA 2 Data Extraction and Annotation Integration link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/All%20of%20Us%20V8%20BRCA%202%20Data%20Extraction%20and%20Annotation%20Integration.ipynb)

### Raw Data Extraction Structuring to Tabular Form - Parsing Script #1
This script was created by Farwa Rizvi, an MSAI student and team member of the BRCA1/2 research project. This script is the first parsing step after data extraction. It converts the raw BRCA1/2 variant data exported from the All of Us V8 pipeline into a structured tabular format. The script cleans and organizes the data, standardizes fields from the annotation tables, and separates complex data values into individual columns. It uses string processing, regular expressions, and token-based parsing to reconstruct and extract key variant features. The result is a readable dataset that can be easily explored, manipulated, and used for downstream analysis and modeling.

[Raw Data Extraction Structuring to Tabular Form - Parsing Script #1 Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/Raw%20Data%20Exrtaction%20Structuring%20to%20Tabular%20Form%20-%20Parsing%20Script%20%231.ipynb)

### PolyPhen-2 Batch Submissions Formatting - Parsing Script #2
This script processes the missense variants in our dataset for batch submission in the PolyPhen-2 prediction tool. Missense variants are a DNA change that substitutes one amino acid for another. The script extracts the amino acid change feature from the dataset and separates it into individual components, including the reference amino acid, position, and alternate amino acid. These values are converted from three-letter amino acid codes to the single-letter format required by PolyPhen-2, as well as the appropriate protein identifier code for BRCA1 and BRCA2 is added. The newly formatted columns are appended to the dataset and exported as an updated file. This step allows hundreds of variants to be submitted to PolyPhen-2 at once, replacing the much slower process of analyzing variants individually.

[PolyPhen Batch-2 Submissions Formatting - Parsing Script #2 Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/PolyPhen-2%20Batch%20Submissions%20Formatting%20-%20Parsing%20Script%20%232.ipynb)

### PolyPhen-2 Results Formatting - Parsing Script #3
This script converts the PolyPhen-2 prediction results from a text file into a structured CSV dataset. Because PolyPhen-2 results cannot be directly downloaded, the output is first copied into a text file and then parsed line by line. The script skips comment lines, splits each row of text into values, and prevents values with two words(like prediction labels: possibly damaging) from misaligning columns. The cleaned data is then organized into a tabular dataset with defined column names and exported as a CSV file. This structured output allows the PolyPhen-2 predictions to be easily merged with the project’s master spreadsheet for further analysis.

[PolyPhen-2 Results Formatting - Parsing Script #3 Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/PolyPhen-2%20Results%20Formatting%20-%20Parsing%20Script%20%233.ipynb)

### BRCA1 Web Models (Polyphen-2 and REVEL) Concordance Script
This script examines the agreement (concordance) between different prediction sources for BRCA1 missense variants. The dataset includes PolyPhen-2 predictions entered by two students, REVEL scores from the Variant Annotation Table (VAT), and clinical classifications from ClinVar. Each variant was predicted in Polyphen-2 by each student to introduce redundancy into our validation, ensuring everyone was getting the same results and minimizing human error. The script first selects the relevant columns and standardizes the prediction labels into common groups (Benign, Disease Causing, and VUS) so they can be compared directly. It then checks that both student PolyPhen-2 submissions match and measures concordance between PolyPhen-2, REVEL, and ClinVar classifications. The final concordance labels are merged back into the main dataset and exported for use in later modeling steps.

[BRCA1 Web Models (Polyphen-2 and REVEL) Concordance Script Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/BRCA1%20Web%20Models%20(Polyphen-2%20and%20REVEL)%20Concordance%20Script.ipynb)

### BRCA2 Web Models (Polyphen-2 and REVEL) Concordance Script
This script performs the same concordance analysis used in the BRCA1 workflow but applies it to BRCA2 variants. Focusing on the same key columns: ClinVar classifications, REVEL scores, and PolyPhen-2 predictions entered by three students. As in the BRCA1 script, prediction outputs from ClinVar, REVEL, and PolyPhen-2 are standardized into common groups (Benign, Disease Causing, or VUS) so they can be compared directly. An added validation step checks that the three student PolyPhen-2 submissions agree with each other before comparing model predictions. Concordance is then evaluated between all 3 prediction sources, with the results used together with BRCA1 for later modeling steps.

[BRCA2 Web Models (Polyphen-2 and REVEL) Concordance Script Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/BRCA2%20Web%20Models%20(Polyphen-2%20and%20REVEL)%20Concordance%20Script.ipynb)

### BRCA 1 and 2 Building Machine Learning Models - Anomaly Script 1
This script builds anomaly detection models, which identify unusual patterns in data that may represent pathogenic variants. The BRCA1 and BRCA2 concordance datasets are combined, and unique identifiers and previous model labels are removed to prepare the data for modeling. The script then explores the dataset, including reviewing variable data types, missing values, and duplicates. Several transformations are applied to make the data model-ready, including converting string-based numeric lists into numeric variables, converting categorical variables (such as amino acids and alleles) into numeric dummy variables, and normalizing the data on a 0 to 1 scale. After all features are converted to numeric format, Isolation Forest and Local Outlier Factor models are trained to detect variants that appear unusual in the dataset. Model results are then compared with the REVEL and PolyPhen-2 classification labels from the concordance analysis to evaluate performance.

[BRCA 1 and 2 Building Machine Learning Models - Anomaly Script 1 Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/BRCA%201%20and%202%20Building%20Machine%20Learning%20Models%20-%20Anomaly%20Script%201.ipynb)

### BRCA 1 and 2 Building Machine Learning Models - Anomaly Script 2
This script builds another anomaly detection model, which again identifies variants that might be pathogenic. Like the first anomaly script, BRCA1 and BRCA2 concordance data is cleaned and prepared by removing unnecessary variables and converting all features into numeric format for modeling. In this version, however, ClinVar significance labels are used to evaluate model performance rather than comparing results to REVEL and PolyPhen-2 predictions. The ClinVar classifications are simplified into two groups, “Disease Causing” and “Other”, to highlight potentially pathogenic variants. The reason for switching the validation labels to Clinvar Significance is to be able to use the REVEL and PolyPhen-2 predictions and their metrics/scores within the data without concern for data leakage. Polyphen-2 and Revel predictions were different enough (only aligned about 65% of the time) to feel comfortable using them as separate features. After preprocessing, Isolation Forest and Local Outlier Factor models are trained to detect variants that appear anomalous, and the results are evaluated based on how well the detected anomalies correspond to ClinVar disease-causing variants.

[BRCA 1 and 2 Building Machine Learning Models - Anomaly Script 2 Link](https://github.com/Nintencode/From-Millions-to-Thousands-Scalable-Genomic-Analysis-of-BRCA1-and-2-Variants/blob/main/BRCA%201%20and%202%20Building%20Machine%20Learning%20Models%20-%20Anomaly%20Script%202.ipynb)

### BRCA 1 and 2 Building Machine Learning Models - Supervised Classification Models
This script builds classification models, which learn patterns from labeled data to predict the category of new variants. Two model types were developed: Random Forest and XGBoost. The BRCA1 and BRCA2 concordance datasets are combined and cleaned again, following a process similar to the anomaly model scripts. ClinVar significance labels are used as the target variable. These labels are simplified into three groups, Disease Causing, Benign, and VUS (Variant of Uncertain Significance), to make prediction easier. The data is then converted into numeric format, split into training, validation, and test sets, and further prepared for machine learning. The goal is to train and fine-tune models that best identify pathogenic and benign variants.

## Pipeline Conclusion
This project developed a scalable computational workflow to analyze BRCA1 and BRCA2 variants in participants with Latin American ancestry from the All of Us Research Program. Using the Hail genomic framework, approximately 45 million whole-genome variants were filtered to the BRCA1/2 genomic regions and reduced to 3,523 ancestry-specific variants. Custom parsing scripts then transformed the raw outputs into structured tabular datasets and automated batch submissions to the web variant prediction model Polyphen-2. These steps replaced previously manual processes, reducing turnaround time while improving the efficiency and reproducibility of the analysis pipeline.

Comparison of prediction tools showed moderate agreement, with PolyPhen-2 and REVEL aligning on about 61% of BRCA1 variants and 66% of BRCA2 variants, while REVEL predictions aligned with non-VUS ClinVar Significance about 92.6% of the time. Multiple anomaly detection and classification models were then developed to explore automated variant prediction. The strongest anomaly model detected pathogenic variants with 75% recall, though overall modeling performance was limited by highly imbalanced data and the absence of key evolutionary conservation and protein structure features. These results highlight both the potential of scalable genomic workflows and the importance of richer biological features for improving personalized variant interpretation.
